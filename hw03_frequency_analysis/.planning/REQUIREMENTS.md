# Requirements: Домашнее задание №3: Частотный анализ

**Defined:** 2026-05-17
**Core Value:** `Top10` должна точно соблюдать формальные правила задания: корректно выделять слова, считать частоты и стабильно сортировать результат.

## v1 Requirements

Requirements for initial release. Each maps to roadmap phases.

### Parsing

- [x] **PARS-01**: Вызывающий код может передать произвольный текст, и `Top10` делит его на слова по любым пробельным символам.
- [x] **PARS-02**: Повторяющиеся пробелы, табы и переводы строк не создают пустых слов в результате подсчета.
- [x] **PARS-03**: `Top10` сохраняет точный токен без нормализации регистра и пунктуации, включая отдельное слово `-`.

### Ranking

- [x] **RANK-01**: `Top10` считает частоту каждого уникального слова ровно по точному значению токена.
- [ ] **RANK-02**: `Top10` сортирует слова по убыванию частоты.
- [ ] **RANK-03**: При одинаковой частоте `Top10` сортирует слова лексикографически.
- [ ] **RANK-04**: `Top10` возвращает не более 10 слов.
- [ ] **RANK-05**: Если на границе top 10 есть несколько слов с одинаковой частотой, `Top10` возвращает лексикографически первые слова, пока не наберется 10 элементов.

### Quality

- [x] **QUAL-01**: Пустой ввод возвращает пустой результат без паники.
- [ ] **QUAL-02**: Поведение `Top10` покрыто юнит-тестами для happy-path и ключевых крайних случаев из README.

## v2 Requirements

Deferred to future release. Tracked but not in current roadmap.

### Optional Normalization

- **NORM-01**: `Top10` игнорирует регистр букв при сравнении слов.
- **NORM-02**: `Top10` не учитывает знаки препинания по краям слова, сохраняя внутренние символы.

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| CLI или отдельное приложение | Задача ограничена пакетной функцией и тестами |
| Морфологическая нормализация | Противоречит базовой формулировке про разные словоформы |
| Микрооптимизация до подтверждения корректности | Для маленькой учебной задачи важнее предсказуемость и читаемость |

## User Stories

- Как проверяющий курса, я хочу вызвать `Top10`, чтобы быстро убедиться, что частотный анализ реализован по формальным правилам.
- Как разработчик решения, я хочу иметь детерминированный результат, чтобы тесты стабильно проходили локально и в CI.

## Acceptance Criteria

- `Top10("")` возвращает пустой слайс.
- Разделение выполняется по whitespace, а не только по одиночному пробелу.
- Регистр и пунктуация в базовой версии сохраняются как часть токена.
- Сортировка идет по частоте, затем по слову.
- Результат ограничен десятью словами.

## Definition of Done

- Все v1 requirements отображены в roadmap.
- Реализация находится в `top.go` и сохраняет контракт `Top10(string) []string`.
- Юнит-тесты в каталоге задания проходят локально.
